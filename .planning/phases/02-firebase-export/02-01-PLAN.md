구현 사항:

3. 각 문서를 아래 필드 구조로 변환한다 (migrate-firebase-orders.ts의 FirebaseItem 인터페이스와 동일):
   - id: doc.id (Firestore 문서 ID — Phase 3에서 firebase_id로 사용)
   - type: d.type ?? null (원본값 "주문"/"반품" 그대로 유지)
   - name: d.name ?? "" (품목명)
   - createdAt: Timestamp -> ISO string 변환 (toDate().toISOString()), null이면 null
   - requester: d.requester ?? null (요청자 이름)
   - requestQty: d.requestQty ?? null (수량+단위 원본 문자열, 예: "2박스")
   - companyNm: d.companyNm ?? null (업체명)
   - orderer: d.orderer ?? null (발주자 이름)
   - orderAt: Timestamp -> ISO string 변환, null이면 null
   - recievedAt: Timestamp -> ISO string 변환, null이면 null (검수일, 오타 "recieved" 유지)
   - confirmQty: d.confirmQty ?? null (확인 수량 원본 문자열)
   - inspector: d.inspector ?? null (검수자 이름)
   - progress: d.progress ?? null (숫자: 0=pending, 1=ordered, 2=inspecting)
   - hasTS: d.hasTS ?? null (거래명세서 수령 boolean)
   - lastEditer: d.lastEditer ?? null (마지막 수정자, 오타 유지)
   - lastEdited: Timestamp -> ISO string 변환, null이면 null
4. Timestamp -> ISO 변환 함수: val이 객체이고 toDate 메서드가 있으면 val.toDate().toISOString() 반환, 아니면 null
5. 출력 경로: src/data/firebase-items.json (디렉토리 없으면 mkdirSync)
6. JSON.stringify(items, null, 2)로 pretty-print 저장
7. 콘솔 출력: 총 문서 수, 날짜 범위 (createdAt 기준 min~max), 출력 파일 경로
