**Per D-03:** `scripts/reset-data.sql` 파일을 생성한다.

SQL 내용:
1. 트랜잭션 시작 (`BEGIN;`)
2. 삭제 전 보존 테이블 건수 스냅샷 (검증용):
   - `SELECT 'profiles' AS tbl, COUNT(*) AS cnt FROM profiles;`
   - `SELECT 'vendors' AS tbl, COUNT(*) AS cnt FROM vendors;`
   - `SELECT 'vendor_products' AS tbl, COUNT(*) AS cnt FROM vendor_products;`
   - `SELECT 'unified_products' AS tbl, COUNT(*) AS cnt FROM unified_products;`
3. 대상 테이블 TRUNCATE (FK 의존성 없으므로 순서 무관, CASCADE 불필요):
   - `TRUNCATE TABLE orders;`
   - `TRUNCATE TABLE blood_records;`
   - `TRUNCATE TABLE activity_logs;`
4. 삭제 후 검증 쿼리:
   - `SELECT 'orders' AS tbl, COUNT(*) AS cnt FROM orders;` (기대값: 0)
   - `SELECT 'blood_records' AS tbl, COUNT(*) AS cnt FROM blood_records;` (기대값: 0)
   - `SELECT 'activity_logs' AS tbl, COUNT(*) AS cnt FROM activity_logs;` (기대값: 0)
5. 보존 테이블 건수 재확인 (삭제 전과 동일해야 함):
   - `SELECT 'profiles' AS tbl, COUNT(*) AS cnt FROM profiles;`
   - `SELECT 'vendors' AS tbl, COUNT(*) AS cnt FROM vendors;`
   - `SELECT 'vendor_products' AS tbl, COUNT(*) AS cnt FROM vendor_products;`
   - `SELECT 'unified_products' AS tbl, COUNT(*) AS cnt FROM unified_products;`
6. `COMMIT;`

상단에 주석으로 설명 추가:
- 목적: 테스트/불일치 데이터 초기화 (Firebase 재마이그레이션 준비)
- 실행 방법: Supabase SQL Editor에서 전체 복사 후 실행
- 주의: profiles, vendors, vendor_products, unified_products는 건드리지 않음
- Per D-01: 백업 불필요 (테스트 데이터)

**Per D-02:** `scripts/reset-storage.ts` 파일을 생성한다.

Storage 정리 스크립트 (`tsx`로 실행 가능):
1. `@supabase/supabase-js` import, `dotenv/config` import
2. `.env.local`에서 `NEXT_PUBLIC_SUPABASE_URL`과 `SUPABASE_SERVICE_ROLE_KEY` 읽기
3. service_role 클라이언트 생성 (`createClient`)
4. `order-photos` 버킷의 전체 파일 목록 조회 (`storage.from('order-photos').list()`)
   - 루트 레벨 폴더 목록 조회 후, 각 폴더 내부 파일도 재귀적으로 조회 (orderId 폴더 구조)
5. 파일이 있으면 배치 삭제 (`storage.from('order-photos').remove(paths)`)
   - Supabase Storage API는 한번에 최대 1000개 삭제 가능 -- 1000개씩 배치 처리
6. 결과 출력: 삭제된 파일 수, 에러 여부
7. 파일이 없으면 "order-photos 버킷이 이미 비어 있습니다" 출력

스크립트 상단에 실행 방법 주석:
```
// 실행: npx tsx scripts/reset-storage.ts
// 필요: .env.local에 NEXT_PUBLIC_SUPABASE_URL, SUPABASE_SERVICE_ROLE_KEY 설정
```
